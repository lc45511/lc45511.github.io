Activity: Database

SQL SELECT Statement

  • SELECT * FROM Customers;

SQL WHERE Clause

  • SELECT * FROM Customers
  • WHERE Country='USA';

SQL COUNT Function

  • SELECT COUNT(ProductID)
  • FROM Products;

SQL INSERT INTO Statement

  • INSERT INTO Customers
  • VALUES ('Cardinal', 'Tom B. Erichsen', 'Skagen 21', 'Stavanger', '4006', 'Norway');

SQL UPDATE Statement

  • UPDATE Customers
  • SET ContactName='Juan';

While completing the SQL activities in this lesson, I learned how databases use SQL commands to retrieve, organize, and modify information efficiently. I gained a better understanding of how the SELECT statement retrieves data from a table, the WHERE clause filters records based on specific criteria, and the COUNT() function summarizes data by counting matching records. I also learned how the INSERT INTO statement adds new records to a database and how the UPDATE statement modifies existing information without affecting the entire table. One important lesson was that the UPDATE statement should always be used carefully because omitting the WHERE clause will update every record in the table instead of only the intended ones. These concepts demonstrated how relational databases manage large amounts of information accurately and efficiently. This knowledge can be applied in accounting by maintaining financial records, tracking transactions, and generating reports. In marketing, SQL can be used to analyze customer data, segment audiences, and measure campaign performance. More generally, understanding SQL provides a valuable foundation for working with business databases, improving decision-making, organizing information, and supporting data-driven operations in many professional environments.
