Activity 1: Complete the following tutorials

The first activity involved completing three tutorials: TutorialsPoint: Internet Reference Models, TutorialsPoint: Internet Protocols, and TutorialsPoint: Internet Domain Name System (DNS). These tutorials provided an overview of how devices communicate across networks and how information is transmitted over the internet. The Internet Reference Models tutorial introduced the OSI and TCP/IP models, explaining how different layers work together to move data between devices. The Internet Protocols tutorial covered protocols such as TCP, IP, UDP, FTP, and HTTP, describing their roles in transmitting data, transferring files, and enabling communication between web browsers and servers. The DNS tutorial explained how domain names are translated into IP addresses, making it possible for users to access websites using easy-to-remember web addresses rather than numerical IP addresses. It also introduced URLs, domain names, DNS servers, and the process used to locate websites on the internet.

Through these activities, I learned that many of the internet services and websites we use every day rely on a complex system of protocols and standards working together behind the scenes. I gained a better understanding of how data is broken into packets, routed across networks, and delivered to its destination. I also learned the difference between reliable protocols such as TCP and faster but less reliable protocols such as UDP. The DNS tutorial was especially interesting because it explained how entering a website address into a browser triggers a process that locates the correct server and retrieves the requested webpage. Before this activity, I understood that websites had domain names and IP addresses, but I did not fully understand how the two were connected.

These concepts can be applied in accounting, marketing, and general business. Many business operations depend on internet-based services, including cloud accounting software, online banking, websites, email systems, and file-sharing platforms. Understanding how these technologies communicate can help professionals troubleshoot issues, make informed technology decisions, and better understand cybersecurity risks. In marketing, knowledge of domains, URLs, and web protocols is useful for managing websites, analyzing web traffic, and improving a company's online presence. More broadly, understanding the infrastructure that supports internet communication provides a stronger foundation for working with modern business technologies and adapting to future technological developments.

Activity 2: Share a folder and connect to the shared folder (macOS)

The second activity involved reviewing Apple's Mac Basics: File Sharing and Connect to Shared Computers and File Servers on a Network tutorials. These tutorials explained how to share folders on a Mac and how to access shared files from another computer on the same network. For this activity, I shared a folder labeled "WEB110 Assignment 4a" from my MacBook Pro and then connected to it from my Intel iMac. The tutorials provided step-by-step instructions for enabling file sharing, selecting folders to share, managing permissions, and connecting to shared resources. They also explained how macOS uses network protocols and user permissions to securely grant access to shared files and folders between devices.

Through this activity, I learned how file sharing works within a network and how multiple computers can securely access the same files. I gained a better understanding of the importance of permissions and access controls, which help determine who can view, modify, or manage shared content. It was helpful to see how easily files can be transferred between devices without relying on external storage devices or cloud services. Successfully connecting to the shared "WEB110 Assignment 4a" folder from my Intel iMac also demonstrated how different computers on the same network can communicate and exchange information efficiently. Personally, I had never used this function before. Most documents I have needed to share have been stored in Google or Microsoft applications, where I could simply grant access through sharing permissions. However, this was a very interesting way to securely share files, especially when additional security or local network access is preferred.

These concepts can be applied in accounting and marketing. In accounting, shared folders can be used to store financial reports, spreadsheets, and supporting documentation that multiple team members need to access. In marketing, teams can share graphics, campaign materials, and project files across devices. More generally, understanding file sharing and network access can improve collaboration, organization, and productivity while maintaining security through proper permission settings. As businesses continue to rely on digital files and remote collaboration, knowing how to manage shared resources will be a valuable skill in any professional setting.
